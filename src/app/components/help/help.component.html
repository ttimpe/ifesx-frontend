01 Netzinfrastruktur & Topologie

REC_ORT

Ortsverzeichnis

VDV 452 Tabelle 4.1

Fundamentale geografische Einheit. Unterscheidet zwischen logischen Bereichen und physischen Haltepunkten.

Feldname Datentyp Beschreibung
ONR_TYP_NR INT(10)

Typisierung des Ortes:

  • 1 Bereich: Logische Klammer (z.B. "Domplatz").
  • 2 Steig: Physischer Mast. Referenziert Typ 1.
  • 9 Betriebshof: Start/Ende für Fahrten.
ORT_NR INT(10) Eindeutige ID (Primärschlüssel). Wertebereich 1..999999.
ORT_NAME VARCHAR(60) Interner Bezeichner. Dient Planungszwecken.
ORT_REF_ORT INT(10) Pflichtfeld für ONR_TYP_NR = 2. Enthält die ORT_NR des übergeordneten Bereichs.
REC_HP

Haltepunkt-Attribute

1:1 Erweiterungstabelle zu REC_ORT (nur für Typ 2). Definiert betriebliche Eigenschaften.

Feldname Datentyp Beschreibung
HP_IFOPT VARCHAR Global Unique ID (IFOPT). Bsp: de:05711:1000:1.
HP_RBL_NR INT Kurznummer für Funk/Bordrechner Anmeldung.
HP_NAME_DRUCK VARCHAR Texte für Innenanzeigen und Ticketdrucker.
REC_SEL

Relationen (Streckenkanten)

Logik: Routing erfolgt gerichtet von Ort A nach Ort B. Für Fahrten in Gegenrichtung ist eine separate Relation erforderlich.
Feldname Datentyp Beschreibung
SEL_VON_ORT INT Start-Steig.
SEL_NACH_ORT INT Ziel-Steig.
SEL_LAENGE INT Distanz (Meter). Wichtig für Odometer und Tarif.
REC_OM

Ortsmarken (Zwischenpunkte)

Punkte ohne regulären Fahrgastwechsel. Dienen der Streckenmodellierung (Polyline) oder als Meldepunkte (LSA).

02 Liniennetz & Routing

REC_LID

Linienstammdaten

Feldname Spezifikation
LI_NR Eindeutige ID (Schlüssel), z.B. 10001.
LI_KUERZEL Anzeigetext, z.B. "SB50".
ROUTEN_NR Nummer für Bordrechner/RBL (z.B. 1 oder 10). Dient der technischen Identifikation der Route.
LID_VERLAUF

Linienverlauf (Routen)

Beschreibt die geordnete Abfolge an Haltestellen für eine Linienvariante (Route).

Feldname Beschreibung
STR_LI_VAR Name der Variante (z.B. "V1-Hbf"). Unterscheidet Hin-/Rückweg.
LFD_LID_NR Laufindex der Haltestelle (1, 2, 3...). Reihenfolge ist zwingend.
ON_BEDARF Flag: Nur Halt bei Bedarf (Haltewunsch).
MENGE_FGR

Fahrzeitgruppen

Gruppiert Fahrzeitprofile (z.B. "HVZ Montag"). Wird der Fahrt (REC_FRT) zugeordnet.

Feldname Bedeutung
FGR_NR ID der Gruppe (z.B. 101).
STR_FGR Kürzel (z.B. "HVZ-MO").

03 Fahrplan & Betrieb

REC_FRT

Fahrtdefinition

Feldname Datentyp Beschreibung
FRT_START INT Abfahrtssekunde ab 00:00.
28800 = 08:00 Uhr.
93600 = 26:00 Uhr (02:00 Folgetag).
FGR_NR INT Verweis auf genutzte Fahrzeitgruppe.
MENGE_BEREICH

Fahrzeuggruppen (Depots)

Klassifiziert Fahrten nach Betriebszweig (z.B. Subunternehmer, Solobus/Gelenkbus Anforderung).

  • BEREICH_NR: ID (z.B. 10).
  • STR_BEREICH: Kürzel (z.B. "GLB" für Gelenkbus).
  • BEREICH_TEXT: Beschreibung (z.B. "Gelenkbusflotte").

04 Fahrgastinformation (FIS)

REC_ZNR

Zielanzeigen (Matrix)

Feldname Datentyp Beschreibung
ZNR_NR INT Zielnummer für IBIS-Gerät (000-999).
ZNR_TEXT VARCHAR(160) Voller Text für die Frontanzeige (z.B. "Hauptbahnhof über City").
SEITENTEXT VARCHAR(160) Optionaler abweichender Text für Seitenanzeigen.
REC_ANR

Ansagenkonfiguration

Verwaltet MP3-Ansagen. Die Zuordnung zur Haltestelle erfolgt im LID_VERLAUF (Feld LID_ANR_NR).

Feldname Beschreibung
ANR_NR Ansagen-ID (z.B. 100).
ANR_DATEI Dateiname der MP3 auf dem Bordrechner (z.B. "ansage_100.mp3").
